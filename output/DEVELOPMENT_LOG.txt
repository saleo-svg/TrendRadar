一、项目背景
═══════════════════════════════════════════════════════════════════════════

1.1 项目简介
    TrendRadar 是开源的多平台热榜监控工具 (github.com/sansan0/TrendRadar)
    原始版本号：v2.3.0
    原始数据源：33+ 平台（微博、知乎、B站、抖音、华尔街见闻、36氪、虎扑、贴吧、百度热搜等）
    数据来源 API：newsnow 开源项目
    用户用途：自动抓取热点新闻，按自定义关键词过滤，生成 HTML 报告

1.2 部署环境
    操作系统：Windows 10 (win32 10.0.19045)
    Shell：PowerShell
    工作目录：F:\TrendRadar
    Python 环境：venv (在 F:\TrendRadar\venv)
    包管理：uv (uv.lock)

[子任务 D-2：解决 push 被拒问题]
    错误 1：GitHub secret scanner 拒绝 push
    原因：原作者 README.md 包含第三方通知服务的 URL 示例占位符
         (格式：`https://<service>/<path>/<TEAM>/<CHANNEL>/<TOKEN>`)
         被 GitHub secret scanner 误判为真实 secret
    错误 2：原作者 master 分支 182 个历史 commit 都含 secret

    解决方案：
    1. 修改 F:\TrendRadar\README.md 中 Webhook URL 占位符
    2. 同样修改 F:\TrendRadar\README-EN.md 中占位符
    3. 重新创建 orphan 分支 fresh-main（丢弃 182 个历史 commit）
    4. 删除 main 分支，重命名 fresh-main 为 main
    5. 最终 push 成功：22dbd954..562caf26 main -> main

[子任务 D-3：创建自定义云端 workflow]
    文件：F:\TrendRadar\.github\workflows\custom-crawler.yml
    关键配置：
        on:
          schedule:
            # UTC 0/4/10/14 = 北京时间 8/12/18/22
            - cron: "0 0,4,10,14 * * *"
          workflow_dispatch:
        # 移除原版 7 天试用过期检查 (Check Expiration step)
        # 读取 secrets：FEISHU_WEBHOOK_URL/TELEGRAM_BOT_TOKEN/etc.
    推送：562caf26 main -> main

[子任务 D-4：手动触发云端试跑]

       https://github.com/saleo-svg/TrendRadar/actions/workflows/custom-crawler.yml
    2. 点 Run workflow
    3. 下拉菜单点绿色 Run workflow 确认
    4. Run #1 成功（completed successfully）
    5. Run 时间：64 秒（13:44:51Z → 13:45:55Z）
    6. Run crawler 实际抓取 34 秒（13:45:06 → 13:45:40）
